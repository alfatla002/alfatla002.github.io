Project

Cloudflare + Nginx Proxy Manager Architecture

Overview

This project focused on designing a reverse proxy and DNS architecture capable of routing multiple services through a single domain while maintaining secure HTTPS connections and clean service separation. The system supports multiple deployed services including APIs, static websites, and cloud-hosted applications.

System Design and Development Process

Cloudflare DNS was configured as the external entry point for all services. Nginx Proxy Manager was deployed internally to route incoming traffic to the appropriate containers or services based on subdomain configuration.

SSL certificates were configured to enforce HTTPS across all services. Cloudflare tunnels were introduced to securely expose locally hosted services without requiring open inbound firewall ports.

The architecture evolved through iterative troubleshooting of routing conflicts, SSL configuration issues, and container networking behavior.

Technologies Used

  • Cloudflare DNS
  • Cloudflare Tunnel
  • Nginx Proxy Manager
  • Docker
  • SSL/TLS configuration

Engineering Challenges & Solutions

Ensuring reliable routing between external DNS and internal containers required careful configuration of proxy rules and network mappings. Debugging SSL issues highlighted the importance of certificate trust chains and proxy headers.

Outcome

The project demonstrates practical infrastructure design, reverse proxy configuration, and real-world deployment workflows used in modern cloud environments.
