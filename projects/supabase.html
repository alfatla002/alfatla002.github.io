Project

Supabase Image Gallery & File Browser

Overview

This project is a cloud-based file management application built on Supabase Storage. The goal was to replicate the usability of modern cloud storage platforms while maintaining secure access control and preventing direct exposure of sensitive storage credentials.

The application allows users to upload, browse, filter, and manage files through a grid-based interface with thumbnail previews and batch operations.

Gallery

System Design and Development Process

Early implementations interacted directly with Supabase Storage from the frontend using the anonymous key. This approach introduced security limitations and restricted control over file operations.

The architecture was redesigned so that all storage interactions were routed through a Node.js backend using the Supabase service role key. The frontend handles authentication and UI interactions, while the backend validates and executes storage operations.

A custom file browser interface was developed to display files in a structured grid layout. Multi-file upload functionality, drag-and-drop interactions, and bulk actions were implemented to improve workflow efficiency.

Deployment involved integrating Cloudflare DNS and Nginx Proxy Manager to securely expose services while keeping backend components isolated.

Technologies Used

  • Supabase Storage and Authentication
  • Node.js backend
  • Vite frontend
  • Cloudflare DNS
  • Nginx Proxy Manager

Engineering Challenges & Solutions

The primary challenge involved balancing usability with security. Direct storage access simplified development but introduced risks. Moving storage logic to the backend allowed stricter validation and improved control.

Handling large numbers of files efficiently required thumbnail rendering and UI optimization to prevent excessive loading times.

Outcome

The project demonstrates practical cloud architecture patterns, secure backend design, and separation of concerns between frontend and infrastructure components.
