Project

Marvel API Microservice

Overview

The Marvel API Microservice is a RESTful backend designed to retrieve, process, and cache data from the Marvel Developer API. The project focused on building a structured backend service capable of handling authenticated external API requests while minimizing unnecessary network calls.

The goal was to demonstrate API integration, backend data processing, and caching strategies commonly used in production applications.

System Design and Development Process

The Marvel API requires MD5-based authentication using timestamped requests. The backend was designed to generate authentication hashes dynamically and manage API communication securely.

Retrieved data was normalized and stored in a Supabase database to reduce repeated API calls and avoid rate-limit issues. API endpoints were implemented to allow filtered retrieval of characters and related data.

Caching logic was introduced so that frequently requested data could be served from the database rather than re-fetching from the external API.

Technologies Used

  • Node.js
  • Supabase database
  • Marvel Developer API
  • MD5 hashing authentication

Engineering Challenges & Solutions

Handling external API rate limits required introducing caching logic and structured data storage. Designing endpoints that balanced flexibility with simplicity required careful planning of request parameters and database schema.

Outcome

The project demonstrates backend service design, third-party API integration, and performance optimization through caching strategies.
