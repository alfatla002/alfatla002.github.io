Project

Java To-Do List & Calendar Application

Overview

This project is a Java-based productivity application designed to manage notes, task lists, checklists, and scheduled activities within a single application. The project focused on applying object-oriented design principles while supporting feature expansion without breaking existing functionality.

System Design and Development Process

The application initially began as a simple note-taking system. As requirements expanded, the architecture was refactored to support multiple independent data types including notes, tasks, and checklist items.

A calendar feature was introduced to associate tasks with due dates, enabling users to track upcoming responsibilities. The internal structure was redesigned to separate data management from user interaction logic, allowing new features to be added safely.

Testing was introduced during refactoring to ensure that newly added features such as multiple note sheets and calendar functionality did not introduce regressions.

Technologies Used

  • Java
  • Object-Oriented Programming
  • Modular class design
  • File-based data handling

Engineering Challenges & Solutions

Expanding functionality while maintaining stability required restructuring class responsibilities and reducing tight coupling between components. This reinforced the importance of modular design and incremental refactoring.

Outcome

The project demonstrates object-oriented design, iterative software development, and structured feature expansion within a growing application.
